# Public vs Local Materials

This note clarifies which CogniPrint materials are intended to be public and which are intended to remain local-only unless explicitly prepared for release.

## Public materials

The following materials may be maintained in the public repository when they are clean and intentionally tracked:

- manuscript source;
- citation metadata;
- software source code;
- public website content;
- provenance policy docs;
- template files;
- cautiously written project summaries;
- selected non-sensitive tracked empirical notes and configs.

## Local-only materials

The following should normally remain local-only unless explicitly reviewed and prepared for release:

- private experimental texts;
- local campaign inputs not intended for public distribution;
- local workspace outputs and temporary run bundles;
- colleague draft notes containing unreviewed claims;
- sensitive or ambiguous source materials.

## Practical rule

If a file contains raw empirical input text, temporary workspace outputs, or material that could be misread as a completed public result, keep it local-only until it is intentionally reviewed and documented.

## Communication rule

When discussing the project publicly, describe the current state using the public tracked materials and cautious empirical wording. Do not present local-only workspace material as if it were already a finalized public study.
