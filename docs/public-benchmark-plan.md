# Public Benchmark Plan

## Why a public benchmark is needed

The current evidence package is useful for manuscript consolidation and colleague review, but it remains corpus-bound and descriptive. A public benchmark is needed to make the next empirical step auditable on openly reviewable materials with explicit provenance and licensing.

## Candidate corpus classes

- public-domain texts;
- permissively licensed texts;
- small manually authored benchmark samples;
- multilingual samples;
- public abstracts only when license and attribution are clear.

## Explicit exclusions

- copyrighted news content without permission;
- private messages;
- raw sensitive texts;
- materials with unclear license data.

## Minimum viable benchmark v1

- `20` to `50` public samples;
- each sample paired with baseline and controlled variants where possible;
- provenance table;
- license field;
- source URL;
- acquisition date.

## Benchmark artifact plan

- `datasets/public-benchmark-v1/`
- `evidence/public-benchmark-v1/`

## Validation outputs

The first public benchmark package should generate:

- benchmark manifest;
- counts;
- methods;
- results;
- limitations;
- provenance.

## Guardrail

This plan defines the next maturity step only. It does not claim that a public benchmark is already complete.
