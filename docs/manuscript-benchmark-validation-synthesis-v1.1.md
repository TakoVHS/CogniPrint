## Current bounded reading

The current manuscript can now say three things more clearly than before.

First, the campaign package is no longer being interpreted in isolation. It is being read against a benchmark-linked descriptive reference layer that is broader than the earlier released subset.

Second, the benchmark-growth layer improves interpretation without removing caution. The move from the smaller `6`-baseline / `36`-variant subset to the broader `9`-baseline / `54`-variant layer makes the random-baseline reference and bridge summaries more informative for reviewer-facing reading.

Third, the combined layer remains manuscript-support material, not an inferential endpoint. The current repository supports discussion of observed patterns, measured shifts, profile differences, perturbation effects, and stability tendencies. It does not support stronger benchmark-generalized or decision-level wording.

## Recommended manuscript phrasing

Use phrasing like:

- "benchmark-linked descriptive validation layer"
- "broader public reference mix"
- "measured perturbation shifts under the current corpus configuration"
- "corpus-bound stability tendencies"

Avoid phrasing like:

- "validated threshold"
- "general benchmark proof"
- "stable across heterogeneous corpora"
- "completed empirical study"

## Practical insertion points

The current manuscript route should use this synthesis in four places:

1. Methods: explain that validation v1.1 is a benchmark-linked descriptive support layer rather than a separate inferential study.
2. Results: contrast the campaign package against the broader random-baseline reference without turning that contrast into a threshold rule.
3. Limitations: state directly that benchmark composition still influences the descriptive frame materially enough to matter.
4. Discussion: treat the combined benchmark-plus-validation layer as reviewer-supporting evidence rather than as a claim of empirical completion.

## Immediate implication

The current manuscript can now be more coherent about why the validation layer is worth including.

It still should not claim that the benchmark-growth layer has stabilized enough to justify stronger inferential framing without external review and further benchmark work.
