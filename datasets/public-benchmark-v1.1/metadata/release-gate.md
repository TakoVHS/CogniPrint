# Public Benchmark v1.1 Release Gate

The next benchmark increment should not be released until all of the following are true.

## Minimum release conditions

- provenance recorded for every released baseline;
- reuse or license note recorded for every released baseline;
- stable source URL recorded for every released baseline;
- acquisition date recorded for every released baseline;
- released variants map cleanly to released baselines;
- released counts match `manifest.json` and `counts.json`;
- no private or local-only source text is included.

## Release recommendation

Prefer a narrower clean release over a larger ambiguous release.

If language or domain balance is still weak but provenance is strong, release the increment only with explicit limitations rather than inflating the claim surface.
